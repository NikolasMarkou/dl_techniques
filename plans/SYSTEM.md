## Identity
`dl_techniques` is a Keras 3 / TensorFlow 2.18 deep-learning research library — 150+ model architectures, 290+ custom layers, plus production training pipelines, applications, and experiments. **Domain: codebase.**

## Boundaries
**In scope** (planner owns): everything under `src/`, `tests/`, `research/`. Library code follows Keras 3 conventions strictly.
**External deps**: tiktoken (gpt2 encoding, n_vocab=50257; 4 reserved specials at 50257..50260), HuggingFace datasets (Wikipedia), tensorflow 2.18.0, keras >=3.8.0 <4.0, numpy, scipy, scikit-learn, matplotlib (always `MPLBACKEND=Agg`).
**Out of scope**: external systems, CI infra. No deployment pipeline lives in this repo.

## Invariants
- **Keras 3 idioms** — `@keras.saving.register_keras_serializable()` on custom layers/models, `keras.ops` (no raw TF), full `get_config()` round-trip, `dl_techniques.utils.logger` (no `print`).
- **Output dict key MUST be `"logits"`** — `MaskedCausalLMLoss` and `model.compile(loss={"logits": ...})` key on it (FINDINGS plan_1519e34f).
- **AdamW WD only** — no `kernel_regularizer=L2(...)` combined with `AdamW(weight_decay=...)` (LESSONS).
- **Causality (causal models)** — `(H=1, W=seq_len)` Clifford blocks must use `avg`/`max` pool only; DSv2 nearest-upsample requires `_causal_upsample` right-shift by `s-1` (LESSONS, plan_82749628 D-007).
- **WaveFieldAttention API locked by 62 tests** — do NOT modify the layer; `(B,N,D)` in/out, `(B,N)` mask, FFT in fp32 under AMP (FINDINGS plan_1519e34f).
- **Pattern-3 CLM CLI uniformity** — every CLM consumer must expose `--steps-per-epoch`, `--seed`, `--min-article-length`, `--shuffle-shards`, `--resume` (D-001..D-006 plan_c6dd7cc1).
- **Frozen tensor state in layers** — must live in `add_weight(trainable=False, ...)` or stored as numpy on `self`; never as a plain tensor created in `build()` (LESSONS — Keras FuncGraph dead-tensor bug).
- **`_generate` decode hardening** — every `GenerationProbeCallback._generate` must mask reserved tiktoken specials at logits time AND wrap final `self._enc.decode(ids)` in `try / except (KeyError, UnicodeDecodeError)` (commit 1fe2088 + plan_08aaf818). Routing variant preserves `Tuple[str, int]` in both branches.
- **CLM compile-time metric floor** — every Pattern-3 CLM trainer's `model.compile` MUST use `metrics={"logits": build_clm_metrics(config.encoding_name)}` (returns `[SparseCategoricalAccuracy, Perplexity, BitsPerToken, BitsPerCharacter]`); never the bare `["accuracy"]` literal. `ignore_index` defaults to `-1` to match `MaskedCausalLMLoss`; pass `ignore_index=-100` for HF-style label padding (plan_3f461682 D-001).
- **`prepare_dict_keyed_compile` is a permanent contract for dict-output trainers** — every CLM trainer that returns a dict from `call()` MUST call `prepare_dict_keyed_compile(model, output_key="logits")` inside `compile_model(...)`, before `model.compile(...)`. Without it, Keras silently drops all `metrics={"logits": [...]}` entries on subclassed models. Helper lives in `train.common.nlp`. Applied to all 6 current CLM trainers (gpt2/pretrain, gpt2/finetune, cliffordnet_nlp, cliffordnet_nlp_unet, cliffordnet_nlp_routing, wave_field_llm/pretrain) in plan_2026-05-07_824e5687. New CLM trainers must wire this from day one (plan_2026-05-07_824e5687 D-001).
- **Probe-time hook contract** — every `GenerationProbeCallback` must expose `_post_generate_hook(self, results: dict) -> None` (default no-op) and call `self._post_generate_hook(probe_results)` inside `_run_probes` after the generation loop. Trainers bind `augment_probe_results` (alias for `aggregate_probe_metrics`) on the probe instance to write `self_bleu` / `distinct_2` / `mean_tok_per_s` into the probe JSONL (plan_3f461682 D-002). Finetune-only trainers (no probe) skip the bind.
- **memory_bank dual-optimizer routing uses leading-component prefix match** — `name.split('/')[0].startswith(p) for p in ("memory_", "gate_"))` in `split_trainable_by_prefix` (called from `WaveFieldMemoryLLM.train_step`). ## Flows
- **CLM training** — `python -m train.<model>.pretrain --config small` → `TrainingConfig` dataclass → tiktoken/Wikipedia dataset → packed token shards → `MaskedCausalLMLoss` on `{"logits": ...}` → AdamW + warmup-cosine → `StepCheckpointCallback` (saves `.keras`) + `GenerationProbeCallback` (samples from a prompt every K steps).
- **CLIP training** — `train.cliffordnet.train_clip` → vision tower (hierarchical CliffordNet) + text tower (tiktoken) → contrastive loss → checkpointing.
- **Generation probe** — during training, sample tokens with greedy/top-k/temperature → mask reserved specials at logits time → decode via tiktoken → log via `dl_techniques.utils.logger`. Decode wrapped in try/except backstop.
- **Save / load round-trip** — `keras.saving.register_keras_serializable()` decorator on every custom class → `.keras` archive → `keras.models.load_model(path, custom_objects={...})` (must include both losses; layers auto-registered).
- **Test scoping** — pytest only on changed module + immediate importers. `make test` reserved for explicit pre-push request (~1.5h).
- **Plan close** — orchestrator writes `summary.md`, runs decision-anchor audit, updates `plans/LESSONS.md` (≤200 lines) and `plans/SYSTEM.md` (≤300 lines), then `bootstrap.mjs close` merges per-plan findings/decisions into `plans/{FINDINGS,DECISIONS}.md` (sliding window: 8 most recent), appends to `plans/INDEX.md`, removes `.current_plan` pointer.

## Known Patterns
- **Sibling-stack model addition** — when a new layer (e.g. `WaveFieldAttention`) doesn't fit the shared attention factory (mask-shape mismatch, novel hyperparam), build a self-contained decoder stack inside the new model package; defer factory registration to a later plan. Trades ~150 LOC duplication for zero blast radius on 30+ unrelated models (plan_1519e34f D-001).
- **Pattern-3 NLP CLM training script** — ~95% generic `TrainingConfig` + `StepCheckpointCallback` + `GenerationProbeCallback` + AdamW/warmup-cosine + tiktoken/Wikipedia + `MaskedCausalLMLoss`. New CLM script = mirror file-by-file, swap model class + variant ladder.
- **Pattern-4 image training script (canonical reference: `src/train/vit/train_vit.py`)** — `@dataclass TrainingConfig` (`model_variant`, `dataset`, `num_classes`, optimizer/schedule/wd/dropout, `success_threshold: Optional[float] = None`) + per-dataset builders (CIFAR via `tf.data` keras-datasets, ImageNet via `image_dataset_from_directory`) + AdamW/SGD WD branch (LESSONS L72) + augment-then-normalize-as-`.map` pipeline + `_assert_train_val_distribution_match` immediately before `model.fit` (CIFAR path) + `MetricsVisualizationCallback` + `train_vit()` returning `{model, best_val_acc, early_stop_epoch, total_epochs, history}` + `main()` guarded SUCCESS-log (`converged AND NOT early-stopped` → INFO, `converged AND early-stopped` → WARNING, `NOT converged` → ERROR + `sys.exit(1)`) + `--success-threshold` CLI override. New image trainer = mirror file-by-file, swap model variant ladder + dataset path. `resnet` and earlier image trainers do NOT yet have the distributional invariant assertion nor the guarded SUCCESS log — forward-looking patterns from plan_2026-05-12_f2d29729 D-007; backport opportunistically.
- **Hierarchical CliffordNet** — multi-stage `CliffordNetBlockDS` (vision) or `CausalCliffordNetBlockDSv2` (language causal) with explicit downsamplers. Stages typically `64→128→256` for ~3M-param CIFAR-100 budget.
- **Pool-before-split for shape-preserving Clifford blocks** — geometric product is element-wise on `(z_det, z_ctx)`; downsample once and feed both streams from the pooled tensor.
- **Mechanical patch replication across N sibling files** — when porting a known-good fix (e.g. 1fe2088 decode hardening), per-file repetition beats shared-helper extraction for N≤4 sites if extraction would cross package boundaries (plan_08aaf818 D-001).
- **DRY metric/loss wiring across N trainers** — when ≥5 trainers share `metrics={"logits": [...]}` shape, factor the *list* into a builder helper in `train.common.nlp` (e.g. `build_clm_metrics`); keep the *math* in `dl_techniques.metrics.<topic>`. Trainer delta becomes a single-line metric swap + 4-line probe-bind. The split survives because the library helper is reusable for non-training callers; the train-side helper centralizes encoding-specific constants (e.g. `_CHARS_PER_TOKEN_DEFAULTS`) and ignore-index propagation (plan_3f461682 D-001).
- **Two-optimizer differential-LR via custom `train_step`** — for memory-augmented / multi-rate models (e.g. `WaveFieldMemoryLLM`, plan_146ae899): register one optimizer with `super().compile(optimizer=...)`, apply the second manually inside `train_step`. Variable routing via name-prefix split (`memory_*` / `gate_*` → memory optimizer; everything else → backbone). Compatible with `prepare_dict_keyed_compile` and `compiled_metrics`. First repo precedent.
- **4-phase curriculum via `Callback` + frozen-trainable-flag flip + KMeans warmup hook** — for memory-augmented models, `PhaseScheduler(keras.callbacks.Callback)` reads `model._global_step` (an `add_weight(trainable=False, dtype=int64)`) inside `on_train_batch_begin`, flips `block.trainable` flags at phase boundaries, and triggers an offline `sklearn.cluster.MiniBatchKMeans` warmup via a model-side `warmup_memory_keys(dataset, num_batches=64)` method at the phase 1→2 boundary. Phase counter (`current_phase`, `add_weight(trainable=False, dtype=int32)`) survives `.keras` save/load round-trip — do NOT serialize via `get_config`. plan_146ae899 SC8/SC9/SC10.

## Codebase Specialization
- **Python**: >=3.11, type hints, Google-style docstrings.
- **Venv**: always `.venv/bin/python` for invocation.
- **Logging**: `from dl_techniques.utils.logger import logger` — never `print`.
- **GPU**: GPU 0 = RTX 4090 24GB, GPU 1 = RTX 4070 12GB. Pin via `CUDA_VISIBLE_DEVICES=N MPLBACKEND=Agg python -m train....` (TF initialises at import, before `setup_gpu()` runs).
- **Test discipline**: scope pytest to touched modules; `make test` only on explicit user request immediately before push.
- **Commit prefix**: `[iter-N/step-M] <description>` during EXECUTE; no commit on EXPLORE/PLAN/REFLECT/PIVOT.
- **Plans dir**: `plans/` is in `.gitignore` (added by bootstrap on `new`); plan files never committed to history.
